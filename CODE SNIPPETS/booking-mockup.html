Fennec FC Trial Booking System

Frontend & Backend Integration Mockup

User Interface - Booking Form

UI Features:

  • Clean, responsive design that matches the club's branding
  • Form fields with appropriate validation cues
  • Clear error messages for validation failures
  • Success message confirmation after submission
Data Flow Architecture
User Input
→
Client Validation
→
Server Validation
→
Database Storage
→
Email Notification

Backend Processing Steps:

  1. Client-side validation ensures basic data integrity
  2. Form data submitted via AJAX to prevent page reload
  3. Server validates inputs using express-validator
  4. Valid submissions stored in MongoDB
  5. Confirmation email sent to applicant
  6. Response with success/error status returned to client
Database Schema
{
  name: {
    type: String,
    required: true
  },
  email: {
    type: String,
    required: true,
    unique: true
  },
  age: {
    type: Number,
    required: true,
    min: 16
  },
  position: {
    type: String,
    required: true,
    enum: ['Goalkeeper', 'Defender', 'Midfielder', 'Forward']
  },
  experience: {
    type: String,
    required: true,
    minlength: 20
  },
  availability: {
    type: String,
    required: true
  },
  submissionDate: {
    type: Date,
    default: Date.now
  },
  status: {
    type: String,
    enum: ['Pending', 'Approved', 'Rejected', 'Completed'],
    default: 'Pending'
  }
}
Email Notification Template

Transactional email with:

  • Club branding for recognition
  • Confirmation of trial booking submission
  • Summary of applicant details
  • Next steps information
  • Contact details for enquiries
Technology Stack
  • Frontend: HTML5, CSS3, Vanilla JavaScript
  • Backend: Node.js with Express
  • Database: MongoDB with Mongoose ODM
  • Form Validation: Client-side JS, Express Validator
  • Email Service: Nodemailer with email provider (e.g., Gmail)
  • Deployment: Potential deployment to Heroku or similar platform
